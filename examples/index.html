Video

new controlize(document.getElementById('video1'));

Animations

There is a problem will the fullscreen api and animations. If you would for some reason use -webkit-animation-fill-mode: both; (as animated.css does) on a parent element to the video, the fullscreen will break.

You could remove the animations after they've finished, or you could add this to your style:

.animated {
   -webkit-animation-fill-mode: none;
   animation-fill-mode: none;
}

Audio

new controlize(document.getElementById('audio1'),{
    playWithSpace: false
});

Methods

Method Description
void draw() Draw the controls. This method is called upon initialization but can be usedful if you've encountered some kind of rendering bug.

Options

Property Description
showControls bool
(default: true) 		Show/Hide the controlpanel for videos
showMuteButton bool
(default video: true)
(default audio: false)
(default touch: false) 		Show/Hide the mute button.
showFullscreenButton bool
(default video: true)
(default audio: false)
(default touch: false) 		Show/Hide the fullscreen button.
showTime bool
(default: true) 		Show/Hide the time elapsed and time left label.
showPlayButton bool
(default: true) 		Show/Hide the play button.
showSeekBar bool
(default: true) 		Show/Hide the seek bar.
playWithSpace bool
(default: true)
(default touch: false) 		Enable/Disable playing the element with the space button. It will only work while the tag is in view, and if more than one tag is in view, they will all play. This option also disables the "page down on space" function when the element is in view.
playWithMouse bool
(default video: true)
(default audio: false)
(default touch: false) 		Enable/Disable the play/pause when clicking a video.
playOnSeek bool
(default: true) 		Play the video/audio after moving the seeker.
hideMouseOnHover bool
(default video: true)
(default audio: false)
(default touch: false) 		Show/Hide the mouse when hovering over a video
